نظرة عامة

انطلق مشروع project-alpha من ملاحظة بسيطة: آلاف المطورين حول العالم يبحثون عن مشاريع مفتوحة المصدر يساهمون فيها، وفي الوقت نفسه يعاني أصحاب المشاريع من إيجاد مساهمين مناسبين. المنصّة تُحلّ هذه المشكلة عبر مطابقة ذكية بين المهارات المتوفّرة لدى المساهم و الاحتياجات الفعلية للمشروع.

تستخدم المنصّة خوارزمية مطابقة تعتمد على تحليل المهارات المُعلَنة في ملفات GitHub للمستخدمين وعلى وسوم المتطلّبات في المستودعات. عند التطابق، يُرسل إشعار لكلا الطرفين مع رابط مباشر للمهمّة المقترحة. النظام يتعامل مع آلاف الطلبات اليومية بزمن استجابة متوسّط أقل من 200 ميلي ثانية.

البنية التقنية

01 / FRONTEND

واجهة Next.js 14

واجهة React مبنية بـ Next.js مع Server Components لتحسين الأداء وSEO، مصمّمة بـ Tailwind CSS.

02 / API LAYER

طبقة tRPC

API مؤمَّن نوعياً (type-safe) بـ tRPC، يربط الواجهة بالخلفية بدون الحاجة لـ REST أو GraphQL.

03 / BUSINESS LOGIC

خوارزمية المطابقة

خدمة Python منفصلة تُحلّل المهارات عبر NLP، تستقبل الطلبات عبر قائمة انتظار Redis.

04 / DATABASE

PostgreSQL + Prisma

قاعدة بيانات علائقية مع Prisma ORM، تضمّ نسخ احتياطية يومية وفهارس محسّنة للأداء.

05 / CACHE & QUEUE

Redis

طبقة تخزين مؤقت للجلسات وقوائم انتظار للمهام غير المتزامنة (إرسال بريد، إشعارات).

06 / DEPLOYMENT

Docker + Vercel

حاويات Docker للخدمات الخلفية، نشر الواجهة على Vercel، CI/CD عبر GitHub Actions.
